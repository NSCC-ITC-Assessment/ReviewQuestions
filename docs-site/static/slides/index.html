The Problem

Code-submission assessments can't answer the fundamental question:

"Did the student actually understand the code they submitted?"
  • Copy-paste and AI-generated code looks identical to authored code
  • Manual code review is time-consuming at classroom scale
  • Plagiarism checkers flag similarity — not comprehension

What is GrillMyCode?

A GitHub Action that automatically analyses a student's code changes and uses AI to generate targeted comprehension questions — ready for a follow-up verbal or written quiz.

🤖
AI-generated questions Tailored to exactly the code the student wrote
⚡
Zero setup for students Runs automatically on every push or PR
🎓
Built for GitHub Classroom Skips template & bot-committed starter files
🔌
Multiple AI providers GitHub Models, OpenAI, OpenRouter, Azure OpenAI

How It Works

  • 1
    Detect the commit range
    Reads the triggering event (push, pull request, etc.) to determine which commits to assess
  • 2
    Collect changed files
    Runs git diff and applies include/exclude glob filters
  • 3
    Strip code comments
    Removes inline and block comments so the AI focuses on logic, not hints
  • 4
    Generate questions via AI
    Sends the cleaned code to the configured AI provider with a structured prompt
  • 5
    Deliver the assessment
    Writes a Markdown file and optionally posts to a PR comment, Issue, or Discussion

Architecture

Docker-based GitHub Action — the runtime is fully pinned and reproducible.

Consumer workflow
      │
      ▼
action.yml  ──► pulls Docker image from ghcr.io
                      │
                      ▼
              entrypoint.sh  ──► cd $GITHUB_WORKSPACE
                                       │
                                       ▼
                                 node src/main.js
                                 │
                    ┌────────────┼────────────────┐
                    ▼            ▼                 ▼
               git.js         ai.js           delivery/
              (diffs)      (provider)    (PR / issue / discussion)

The Docker image is built once and reused — consumer repos pay no build cost at runtime.

Comment Stripping

Before

# Add two numbers together
def add(a, b):
    # Return the sum
    return a + b

After

def add(a, b):
    return a + b

The rmcm binary removes comments before sending to the AI. This forces questions to target the logic, not hints left by the student in their comments.

Can be disabled with keep_comments: 'true'. Falls back gracefully for unsupported file types.

Delivery Options

The assessment is always written to a Markdown file. Any combination of the options below can be enabled at the same time.

💬
PR Comment Posted inline on the pull request. On by default.
🐛
GitHub Issue Auto-assigned to the student. Updated in place on re-runs.
🗣️
Discussion Forum-style record. Discussions are auto-enabled if needed.
📄
Markdown File Committed to .assessment/ in the repo.

AI Providers

GitHub Models default github-models No Uses built-in GITHUB_TOKEN
OpenAI openai Yes GPT-4o, GPT-4 Turbo, etc.
OpenRouter openrouter Yes Access to many open models

All providers share the same OpenAI-compatible chat completions API shape — swapping is a config change only.

GitHub Classroom Integration

GrillMyCode is designed to work seamlessly with GitHub Classroom assignment repositories.

  • Skips starter/template files — pins the diff base to the first student commit so instructor-provided boilerplate is never assessed
  • Filters bot commits — leading commits by github-classroom[bot] and github-actions[bot] are automatically excluded
  • Assignment context — pass the brief, rubric, or README as assignment_context so the AI asks questions relevant to the assignment goals
  • Auto-assigns issues — GitHub Issues are assigned directly to the student who authored the head commit

Key Customisation Inputs

InputDefaultWhat it does
num_questions5How many questions to generate (1–50)
include_answersfalseAppend model answers for an instructor copy
additional_context—Free-text instructor instructions injected into the prompt
assignment_context—Globs for brief/rubric files to inject into the prompt
include_patterns—Limit assessment to specific file globs
exclude_patternsautoSkip non-code files (lock files, images, etc.)
ai_temperature0.5Randomness of generated questions (0 = deterministic)

Quick Start

Add this file to the student repository — no secrets needed.

# .github/workflows/grill-my-code.yml
name: Code Assessment
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  generate-questions:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write   # post the PR comment
      models: read           # GitHub Models access
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0     # full history required for diff resolution

      - uses: NSCC-ITC-Assessment/GrillMyCode@v1
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          num_questions: '5'
          additional_context: 'Assignment 3 – Python list comprehensions'

What the Student Sees

A PR comment appears automatically after every push:

🔥 Code Assessment

1. Your add() function uses positional parameters — what would happen if a caller passed keyword arguments in the wrong order?

2. The loop on line 12 iterates over a list. Why did you choose a for loop rather than a while loop here?

3. …

A Markdown file is committed to the repo:

.assessment/
  └── 2024-11-15-main.md

Optionally, a GitHub Issue or Discussion is also created and updated in place on subsequent submissions.
